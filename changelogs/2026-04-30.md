# 📅 2026-04-30 (목) 변경 이력

## 🚀 주요 업데이트

### 1. Phaser.js 게임 엔진 통합 (Game Boilerplate 전환)
기존의 강력한 네트워크 및 보안 인프라를 유지하면서, UI 레이어를 Phaser.js로 교체하여 게임 개발에 최적화된 보일러플레이트로 전환했습니다.
- **Phaser 3.x 설치**: 고성능 2D 게임 개발을 위한 업계 표준 엔진 도입.
- **게임 브랜치(`game`) 분리**: 기존 대시보드 구조와 격리된 독립적인 게임 개발 환경 구축.
- **멀티플레이어 기반 마련**: DI 컨테이너를 통해 게임 씬 내부에서 `SocketClient`를 직접 주입받아 사용할 수 있는 구조 설계.

### 2. 렌더러 아키텍처 및 진입점 재구성
- **진입점(`main.ts`) 신규 구축**: 누락되었던 렌더러 진입점을 생성하고 Phaser 엔진의 라이프사이클과 부트스트랩 로직을 연결.
- **Phaser 래퍼 클래스 구현**: `PhaserGame.ts`를 통해 게임 설정 및 인스턴스를 중앙 관리.
- **인프라 연동형 메인 씬(`MainScene.ts`)**: 물리 엔진(Arcade)과 애니메이션이 포함된 기본 씬을 구축하고 소켓 연결 확인 로직 포함.

### 3. 레거시 UI 및 미사용 인프라 제거 (Cleanup)
Phaser 중심의 게임 개발 환경을 위해 당장 사용하지 않는 기존 렌더러 기능과 Node.js 전용 네트워크 모듈을 정리했습니다.
- **렌더러 최적화**: 기존 대시보드용 컨트롤러, 뷰, 라우터, 로거 서비스를 삭제하고 `index.html`에서 로그 패널 및 UI 오버레이를 제거했습니다.
- **네트워크 인프라 간소화**: 웹과의 호환성을 위해 TCP(Client/Server) 및 UDP 모듈을 삭제하고, 관련 IPC 핸들러 및 Preload 브릿지를 정리했습니다.
- **스타일 시트 경량화**: `components.css`를 게임에 필요한 최소한의 스타일로 재작성했습니다.

### 4. 게임 개발 가이드 및 문서화
- **PHASER_GUIDE.md**: Phaser 씬 개발 규약, DI 서비스 연동 방법, 멀티플레이 구현 예시 등 상세 가이드 작성.
- **README.md 갱신**: 게임 개발 보일러플레이트 사양에 맞춰 프로젝트 소개 및 실행 방법 업데이트.
- **NETWORK_GUIDE.md 갱신**: 제거된 TCP/UDP 내용을 삭제하고 HTTP/Socket.io 중심으로 재편.
