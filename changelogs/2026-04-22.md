# 📝 개발 로그 (2026-04-22)

## 🚀 오늘의 주요 성과
- **렌더러 아키텍처 대규모 리팩토링**: 
    - `UIRouterService`에 집중되어 있던 HTML 템플릿을 도메인별 **`View` 클래스**로 완전 분리하였습니다.
    - 컴포넌트 기반 설계를 통해 코드 가독성을 높이고 'God Object' 문제를 해결했습니다.
- **OS 통합 기능 구현 완료**: 시스템 트레이, 네이티브 알림, 전역 단축키를 앱 전체에 통합 및 UI와 연결하였습니다.
- **보안 및 설정 영구 저장 강화**: 
    - 암호화 알고리즘을 **AES-256-GCM**으로 업그레이드하여 데이터 무결성 검증 기능을 추가했습니다.
    - 테마 및 언어 설정을 암호화하여 로컬 디스크에 저장하고 앱 시작 시 자동 복원하는 기능을 구현했습니다.
- **다국어(i18n) 인프라 구축**: 한국어/영어 지원을 위한 언어 팩 및 `UISettingsService`를 구현했습니다.
- **Windows 알림 호환성 확보**: `AppUserModelId` 설정을 통해 Windows 환경에서도 알림이 정상 작동하도록 조치하였습니다.

## 🛠 상세 변경 내역

### 1. 렌더러 View-Controller 분리
- `NetworkView`, `DeviceView`, `SharedView`, `LoggerView`, `MaintenanceView` 클래스 신설.
- `UIRouterService`가 주입받은 View 객체에 렌더링 업무를 위임하도록 구조 변경.
- 단일 파일 집중 현상을 해소하고 도메인별 코드 응집도 향상.

### 2. 보안 및 설정 인프라 고도화
- `PersistenceService`를 리팩토링하여 AES-256-GCM 모드 적용 및 Auth Tag 연동.
- 암호화 키 관리 로직 개선 (32바이트 Buffer 강제 조정).

### 2. UI 설정 영구 저장
- `UISettingsService`가 `electronAPI.persistence`를 통해 설정을 비동기로 로드하도록 개선.
- 앱 시작(`bootstrap`) 시 사용자의 테마 및 언어 설정을 암호화 저장소에서 복원.

### 3. OS Core 및 IPC 통합
- `src/main.ts`에 `persistence-set`, `persistence-get` IPC 핸들러 추가.
- 창 닫기('X') 시 트레이로 숨기기 및 단축키 연동 안정화 (Destroyed 에러 방지).
