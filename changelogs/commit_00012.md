# 🚀 COMMIT_00012: LLM 에이전트 통합, 스트리밍, 세션 관리 및 시스템 안정화

## 🚀 주요 업데이트
- **LLM 에이전트(Agentic Workflow) 구현**: LLM이 파일 시스템 조작 도구(READ/WRITE/LIST/DELETE)를 사용하여 직접 코드를 작성하고 파일을 관리할 수 있는 ReAct 루프를 구현했습니다. 외부 경로 접근 권한을 동적으로 관리할 수 있습니다.
- **응답 스트리밍(Streaming) 도입**: 생성된 텍스트 청크를 즉시 렌더러로 전송하여 응답 속도 체감 및 긴 답변 끊김 현상을 방지했습니다.
- **대화 세션 관리 시스템**: 대화 내용 및 모델 기억(Context)을 로컬 디스크에 암호화/압축 저장하여 세션 유지 기능을 구현했습니다. 여러 대화 방 생성 및 전환이 가능합니다.
- **시스템 설정 강화**: `n_ctx` 컨텍스트 길이 설정 기능 추가 및 서버 포트 변경(8080 -> 8888)을 통해 네트워크 충돌을 해결했습니다.
- **에이전트 제어 강화**: 사용자 명령 중단(Abort), 문맥 초기화 기능 및 실시간 상태 피드백 UI(상태 배지, 작업 진행률 표시)를 구현했습니다.
- **프로세스 안정성 확보**: 모델 변경 및 재시작 시 기존 프로세스를 안전하게 종료하고 포트 충돌을 방지하도록 로직을 개선했습니다.
