# CHANGELOG - v1.1.0 (리팩토링 및 기능 확장)

본 문서에서는 `initial-state` 브랜치에서 진행된 아키텍처 리팩토링 및 신규 기능 구현 내역을 기록합니다.

## 🚀 주요 변경 사항

### 1. 아키텍처 리팩토링
- **폴더 구조 재구조화**: 기능별 모듈화를 위해 `src/main/features`와 `src/renderer/features`를 도메인 단위로 분리했습니다.
- **백엔드 모듈화**: `BackendModule` 인터페이스를 통해 백엔드 피처들이 `MainRegistry`에 자동으로 등록되도록 개선했습니다.
- **IPC 통신 통합**: 모듈 간 통신 및 렌더러와의 IPC 통신을 위해 `Bridge` 패턴을 적용했습니다.

### 2. 신규 기능 구현
- **OS 기능**: 알림(Notify), 단축키(Shortcut), 트레이 관리 기능 이식.
- **시스템 정보**: 시스템 상태(CPU, 메모리 등) 모니터링 기능 추가.
- **데이터 지속성(Persistence)**: AES-256-GCM 암호화 및 데이터 압축을 통한 안전한 설정 저장소 구축.
- **보안(Security)**: AES, RSA, Gzip 압축 테스트 유틸리티 및 전용 테스트 모듈 구현.
- **시리얼 통신**: 시리얼 포트 제어 및 실시간 데이터 입출력 기능 통합.
- **미디어 및 장치**: 카메라/마이크 접근, 게임 컨트롤러 제어 및 장치 선택 권한 관리 구현.
- **파일 시스템**: 로컬 파일 시스템 읽기/쓰기 및 파일 다이얼로그 기능 구현.
- **로깅 시스템**: 사용자 선택 경로에 시스템 로그를 기록하는 관리 서비스 구현.

### 3. UI/UX 개선
- **대시보드 개편**: 카드 스타일의 인터랙티브 대시보드로 UI 전면 개편.
- **스타일 시스템**: 메인 브랜치의 스타일 시스템(`main.css`) 이식 및 다크 테마 일관성 적용.
- **Lucide 아이콘**: 시각적 가독성 향상을 위해 Lucide 라이브러리 도입.
- **반응형 개선**: 모바일 환경과 데스크탑 환경을 고려한 레이아웃 보완 및 뒤로가기 버튼 가시성 확보.
- **플랫폼 검증**: 환경 감지(`isElectron`)를 통해 웹/데스크탑 환경에 따른 기능 접근 제한 로직 추가.

### 4. 기타
- **빌드 시스템**: `concurrently` 없이 빌드 체인을 통합하고 `preload` 스크립트를 `.cjs`로 번들링하여 ESM 로딩 에러 해결.
- **문서화**: 프로젝트 설정 가이드(`docs/CONFIGURATION_GUIDE.md`) 추가.
