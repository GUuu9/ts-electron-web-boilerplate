# 🚀 Changelog: 2026-05-12 (화)

## 🌟 주요 업데이트

### 🏗 Main Process 모듈러 아키텍처 (Plug-and-Play) 도입
메인 프로세스의 유지보수 편의성을 극대화하기 위해, 기능을 한 줄의 코드로 추가하거나 삭제할 수 있는 **모듈러 시스템**을 구축했습니다.

- **Main Process 모듈화 (CoreRegistry)**:
  - `CoreRegistry`를 도입하여 메인 프로세스의 IPC 핸들러와 라이프사이클을 도메인별로 격리했습니다.
  - `main.ts`는 더 이상 수십 개의 핸들러를 직접 가지지 않으며, 각 피처가 자신의 핸들러를 스스로 관리하는 구조로 변경되었습니다.

- **Preload Bridge 모듈화**:
  - `preload.ts`의 거대한 API 객체를 도메인별 Bridge 파일로 분리했습니다.
  - Spread 연산자를 사용하여 필요한 API 세트만 선택적으로 노출할 수 있는 유연한 구조를 확보했습니다.

- **계층 및 폴더 구조 재편 (Infra vs Feature 분리)**:
  - **`src/core` (Infra)**: DI, Bridge, Registry 등 앱의 순수 엔진 로직만 남겼습니다.
  - **`src/features` (Feature)**: Network, Device 등 실제 메인 프로세스용 비즈니스 로직들을 이곳으로 모아 관리 포인트를 명확히 했습니다.

## 🛠 수정된 파일

### **Infrastructure (Engine)**
- `src/core/core-feature.ts`: 메인 프로세스 모듈 규격 정의
- `src/core/core-registry.ts`: 메인 프로세스 모듈 등록 및 관리 엔진
- `src/core/bridge/`: 도메인별 Bridge 파일들 생성 및 관리

### **Features (Domain Logic)**
- `src/features/`: 모든 메인 프로세스 도메인 로직 이동 및 `*.core.ts` 구현

### **Entry Points & Docs**
- `src/main.ts`, `src/preload.ts`: 모듈 등록 방식의 간결한 코드로 전면 개편
- `README.md` 및 각종 `*_GUIDE.md`: 새로운 아키텍처와 폴더 구조에 맞춰 업데이트
