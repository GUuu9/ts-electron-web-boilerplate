# 🚀 Changelog: 2026-05-06 (수)

## 🌟 주요 업데이트

### 🖥 UI Logger 고도화 (플로팅 & 도킹 시스템)
사용자가 앱 어디서나 로그를 확인하고 시스템을 제어할 수 있도록 **인터랙티브 로깅 시스템**을 대폭 강화했습니다.

- **플로팅 인터페이스 (Floating UI)**:
  - 로그 패널을 화면 어디로든 **드래그(Drag)**하여 이동할 수 있습니다.
  - 우측 하단 핸들을 통해 창의 **크기(Resize)**를 자유롭게 조절할 수 있습니다.
  - 화면 경계값을 감지하여 창이 화면 밖으로 나가지 않도록 **Boundary Clamping** 로직을 적용했습니다.
  - **최소화(Minimize)** 기능을 통해 작업 공간을 효율적으로 활용할 수 있습니다.

- **도킹 모드 (Docking Mode)**:
  - 로그 창을 하단에 고정하여 안정적으로 모니터링할 수 있는 모드를 추가했습니다.
  - 도킹 시 메인 콘텐츠가 가려지지 않도록 **동적 레이아웃 패딩(Padding)** 로직을 적용했습니다.
  - **단축키 (`Cmd/Ctrl + Shift + L`)** 또는 커맨드(` /logger-mode`)로 즉시 전환이 가능합니다.

- **외부 창 분리 (External Window Detach)**:
  - **데스크탑 전용**: 버튼 하나로 로그 창을 별도의 OS 윈도우로 분리할 수 있습니다.
  - **양방향 커맨드 릴레이**: 분리된 외부 창에서도 명령어를 입력하면 메인 프로세스를 거쳐 메인 창에서 실행되고, 결과 로그가 다시 외부 창으로 전달되는 완전한 양방향 통신 구조를 구축했습니다.

## 🛠 수정된 파일

### **Main Process (Desktop Core)**
- `src/main.ts`: 외부 로그 창 생성을 위한 `BrowserWindow` 관리 및 명령어 릴레이 IPC 핸들러 추가
- `src/preload.ts`: 외부 창과의 통신을 위한 `electronAPI.logger` 브릿지 확장

### **Renderer Process (UI)**
- `src/renderer/index.html`: 플로팅 로그 창 구조 및 제어 버튼(최소화, 분리) 추가
- `src/renderer/logger.html`: 외부 분리 창 전용 UI 페이지 구현
- `src/renderer/src/core/ui-logger.service.ts`: 드래그, 리사이즈, 도킹, 분리 상태 관리 및 마우스 이벤트 핸들링 로직 구현
- `src/renderer/src/features/tester/tester.controller.ts`: 외부 입력 명령어를 처리할 수 있도록 실행 로직 리팩토링
- `src/renderer/src/features/tester/commands.ts`: `/logger-mode` 명령어 추가
- `src/renderer/src/styles/components.css`: 플로팅/도킹 스타일 및 애니메이션 정의
- `src/renderer/src/main.ts`: 외부 명령어 수신 리스너 등록 및 초기화 로직 보완

### **Infrastructure & Docs**
- `vite.config.ts`: 멀티 페이지(`index.html`, `logger.html`) 빌드를 위한 설정 업데이트
- `README.md`, `src/core/logger/LOGGER_GUIDE.md`, `src/renderer/RENDERER_GUIDE.md`: 신규 기능 가이드 반영
